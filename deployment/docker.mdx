## Overview

The Docker Compose configuration provides containerized PostgreSQL and Redis services. The Next.js application runs on the host machine, connecting to these services.

<Note>
  This is a hybrid approach: databases run in containers while the Node.js app runs natively. For full containerization, extend the `docker-compose.yml` to include a web service.
</Note>

## Prerequisites

- Docker Engine 20.10+
- Docker Compose v2+
- Node.js 20+
- pnpm (or npm)

## Docker Compose Configuration

The `docker-compose.yml` file defines two services:

```yaml docker-compose.yml
services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: app_dev
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    ports:
      - "6379:6379"
    volumes:
      - redisdata:/data

volumes:
  pgdata:
  redisdata:
```

## Deployment Steps

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone <repository-url>
    cd smyeg-app
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Start Docker Services">
    Launch PostgreSQL and Redis in detached mode:
    
    ```bash
    docker compose up -d
    ```
    
    Verify services are running:
    
    ```bash
    docker compose ps
    ```
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the project root:
    
    ```bash .env
    DATABASE_URL="postgresql://postgres:postgres@localhost:5432/app_dev"
    REDIS_URL="redis://localhost:6379"
    AUTH_SECRET="your-generated-secret"
    NEXTAUTH_URL="http://localhost:3000"
    ```
    
    <Tip>
      Generate `AUTH_SECRET` with: `openssl rand -base64 32`
    </Tip>
  </Step>

  <Step title="Initialize Database">
    Generate Prisma client and run migrations:
    
    ```bash
    npm run db:generate
    npm run db:migrate
    ```
    
    Seed initial data (creates admin user and base records):
    
    ```bash
    npm run db:seed
    ```
  </Step>

  <Step title="Start the Application">
    For development:
    
    ```bash
    npm run dev
    ```
    
    For production:
    
    ```bash
    npm run build
    npm start
    ```
  </Step>

  <Step title="Start Background Workers">
    In separate terminals, launch the workers:
    
    ```bash
    # Geo worker
    pnpm worker:geo
    
    # Asset worker
    pnpm worker:assets
    ```
    
    See [worker documentation](/deployment/workers/geo-worker) for configuration options.
  </Step>
</Steps>

## Service Management

### View Logs

```bash
# All services
docker compose logs -f

# PostgreSQL only
docker compose logs -f postgres

# Redis only
docker compose logs -f redis
```

### Stop Services

```bash
# Stop without removing containers
docker compose stop

# Stop and remove containers (data persists in volumes)
docker compose down
```

### Restart Services

```bash
docker compose restart
```

## Data Persistence

Docker volumes ensure data survives container restarts:

- `pgdata`: PostgreSQL database files
- `redisdata`: Redis persistence files (AOF enabled)

### Backup Database

```bash
docker compose exec postgres pg_dump -U postgres app_dev > backup.sql
```

### Restore Database

```bash
docker compose exec -T postgres psql -U postgres app_dev < backup.sql
```

## Production Considerations

<Warning>
  The default configuration is for development. For production, make these changes:
</Warning>

### Secure Credentials

Change default passwords in `docker-compose.yml`:

```yaml
environment:
  POSTGRES_USER: smyeg_user
  POSTGRES_PASSWORD: <strong-random-password>
  POSTGRES_DB: smyeg_production
```

Update `DATABASE_URL` in `.env` accordingly.

### Restrict Port Access

Remove port mappings to prevent external access:

```yaml
# Before (development)
ports:
  - "5432:5432"

# After (production)
expose:
  - 5432
```

The app container can still connect via Docker network.

### Enable TLS for Redis

For sensitive session data, configure Redis with TLS and authentication:

```yaml
redis:
  image: redis:7-alpine
  command: redis-server --requirepass <redis-password> --appendonly yes
```

### Resource Limits

Add resource constraints:

```yaml
postgres:
  deploy:
    resources:
      limits:
        cpus: '2'
        memory: 2G
```

## Troubleshooting

### Port Already in Use

**Error**: `Bind for 0.0.0.0:5432 failed: port is already allocated`

**Solution**: Change the host port mapping:

```yaml
ports:
  - "5433:5432"  # Use port 5433 on host
```

Update `DATABASE_URL` to `postgresql://...@localhost:5433/...`

### Connection Refused

**Error**: `ECONNREFUSED 127.0.0.1:5432`

**Solution**: Ensure containers are running:

```bash
docker compose ps
```

If status is "Exited", check logs:

```bash
docker compose logs postgres
```

### Volume Permission Issues

On Linux, if PostgreSQL fails to start with permission errors:

```bash
sudo chown -R 999:999 /var/lib/docker/volumes/smyeg_pgdata
```
