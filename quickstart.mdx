This guide will help you set up SMyEG and get your first organization running quickly.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 20+** - [Download here](https://nodejs.org/)
- **Docker** (recommended) - [Download here](https://www.docker.com/)

## Setup Steps

<Steps>
  <Step title="Install dependencies">
    Clone the repository and install dependencies:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Start Docker services">
    Start PostgreSQL and Redis using Docker Compose:

    ```bash
    docker compose up -d
    ```

    This starts:
    - PostgreSQL 16 with PostGIS extensions (port 5432)
    - Redis 7 (port 6379)
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root with the following variables:

    ```bash .env
    # Database
    DATABASE_URL="postgresql://postgres:postgres@localhost:5432/smyeg"

    # NextAuth
    NEXTAUTH_SECRET="your-secret-key-here"
    NEXTAUTH_URL="http://localhost:3000"

    # Node Environment
    NODE_ENV="development"
    ```

    <Warning>
    Generate a secure `NEXTAUTH_SECRET` using:
    ```bash
    openssl rand -base64 32
    ```
    </Warning>
  </Step>

  <Step title="Generate Prisma client">
    Generate the Prisma client from the schema:

    ```bash
    npm run db:generate
    ```
  </Step>

  <Step title="Run database migrations">
    Apply database migrations to create tables and extensions:

    ```bash
    npm run db:migrate
    ```

    This creates:
    - All database tables from the Prisma schema
    - PostGIS extensions for geospatial data
    - Partitioned audit log tables
    - Database triggers for geometry calculations
  </Step>

  <Step title="Seed the database">
    Load initial data including the admin user and default organization:

    ```bash
    npm run db:seed
    ```

    This creates:
    - Default organization: "Por defecto"
    - System modules and permissions
    - Admin role with full permissions
    - Admin user account
  </Step>

  <Step title="Start the development server">
    Run the Next.js development server:

    ```bash
    npm run dev
    ```

    The application will be available at [http://localhost:3000](http://localhost:3000)
  </Step>

  <Step title="Login and create your first organization">
    1. Navigate to [http://localhost:3000](http://localhost:3000)
    2. Login with the default admin credentials:
       - **Email**: `admin@example.com`
       - **Password**: `Admin1234`
    
    <Warning>
    Change the admin password immediately after first login for security.
    </Warning>

    3. Once logged in, navigate to **Organizations** in the sidebar
    4. Click **Create Organization** and fill in:
       - Organization name
       - RIF (tax ID)
       - Country (optional)
    
    5. Your new organization is now ready to use!
  </Step>
</Steps>

## Optional: Start Background Workers

For geospatial import processing and asset measurements, start the background workers:

<Accordion title="Geospatial worker">
  Process shapefile imports and geometry calculations:

  ```bash
  npm run worker:geo
  ```

  Or run once for testing:
  ```bash
  npm run worker:geo:once
  ```
</Accordion>

<Accordion title="Asset measurement worker">
  Process asset measurement calculation jobs:

  ```bash
  npm run worker:assets
  ```

  Or run once for testing:
  ```bash
  npm run worker:assets:once
  ```
</Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection refused">
    Ensure Docker is running and PostgreSQL container is up:
    ```bash
    docker compose ps
    docker compose logs postgres
    ```
  </Accordion>

  <Accordion title="Migration errors">
    Reset the database and rerun migrations:
    ```bash
    docker compose down -v
    docker compose up -d
    npm run db:migrate
    npm run db:seed
    ```
  </Accordion>

  <Accordion title="Port 3000 already in use">
    Stop any process using port 3000 or change the port:
    ```bash
    # On Linux/Mac
    lsof -ti:3000 | xargs kill -9
    
    # On Windows
    netstat -ano | findstr :3000
    taskkill /PID <PID> /F
    ```
  </Accordion>
</AccordionGroup>
