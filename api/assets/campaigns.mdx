## Overview

AssetMeasurementCampaign endpoints manage field measurement campaigns across four domains: biological (survival, pre-harvest, continuous monitoring), hydrological (meteorological and hydrological stations), dendrometric (carbon, multipurpose plots), and demographic (community and household surveys).

## Campaign Status Enum

<ResponseField name="status" type="enum">
  - `DRAFT` - Campaign in preparation
  - `ACTIVE` - Campaign in progress
  - `CLOSED` - Campaign completed
  - `CANCELLED` - Campaign cancelled
</ResponseField>

## Asset Domain Enum

<ResponseField name="domain" type="enum">
  - `BIOLOGICO` - Biological assets monitoring
  - `HIDROLOGICO` - Hydrological monitoring
  - `DENDROMETRICO` - Dendrometric measurements
  - `DEMOGRAFICO` - Demographic surveys
</ResponseField>

## Measurement Protocol Types

<ResponseField name="protocolType" type="enum">
  **Biological Domain:**
  - `BIO_SOBREVIVENCIA` - Survival inventory
  - `BIO_PREOPERATIVO` - Pre-operational assessment
  - `BIO_PRECOSECHA` - Pre-harvest evaluation
  - `BIO_CONTINUO` - Continuous monitoring
  - `BIO_PARCELAS_PERMANENTES` - Permanent plots

  **Hydrological Domain:**
  - `HIDRO_ESTACION_METEOROLOGICA` - Meteorological station
  - `HIDRO_ESTACION_HIDROLOGICA` - Hydrological station

  **Dendrometric Domain:**
  - `DENDRO_PARCELA_MULTIPROPOSITO` - Multipurpose plots
  - `DENDRO_PARCELA_CARBONO` - Carbon plots

  **Demographic Domain:**
  - `DEMO_ENCUESTA_COMUNIDAD` - Community survey
  - `DEMO_ENCUESTA_NUCLEO_FAMILIAR` - Household survey
</ResponseField>

<Note>
  Each protocol type has specific validation rules and calculation formulas implemented in the worker.
</Note>

## DELETE /api/assets/measurement-campaigns

Delete a campaign. Only campaigns without observations or results can be deleted.

### Request Body

<ParamField body="id" type="string" format="uuid" required>
  Campaign ID
</ParamField>

### Response

Returns the deleted campaign ID.

### Error Responses

- **404** - Campaign not found
- **409** - Campaign has associated observations or results and cannot be deleted

<RequestExample>
```bash cURL
curl -X DELETE "https://api.smyeg.com/api/assets/measurement-campaigns" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"id": "a1b2c3d4-5678-90ab-cdef-123456789abc"}'
```
</RequestExample>

## Import Endpoints

### GET /api/assets/measurement-campaigns/import

Download import template in CSV or Excel format.

<ParamField query="format" type="enum" default="csv">
  Template format: csv, xlsx
</ParamField>

### POST /api/assets/measurement-campaigns/import

Bulk import campaigns from CSV, Excel, or JSON file.

**Supported Formats:**
- `.csv` - Comma-separated values
- `.xlsx` - Excel spreadsheet
- `.json` - JSON array

**CSV/Excel Column Headers:**
- `level4Id` - Level 4 ID (UUID)
- `domain` - Asset domain
- `protocolType` - Protocol type
- `campaignCode` - Campaign code
- `campaignName` - Campaign name
- `campaignDate` - Campaign date (YYYY-MM-DD)
- `status` - Status (optional, defaults to DRAFT)
- `schemaVersion` - Schema version (optional, defaults to 1.0)
- `notes` - Notes (optional)

### Response

<ResponseField name="created" type="integer">
  Number of campaigns created
</ResponseField>

<ResponseField name="skipped" type="integer">
  Number of rows skipped
</ResponseField>

<ResponseField name="errors" type="array">
  Array of error objects with row number and error message
</ResponseField>

## Permissions

- **READ** - View campaigns (permission: `asset-measurement:READ`)
- **CREATE** - Create campaigns and import data (permission: `asset-measurement:CREATE`)
- **UPDATE** - Modify campaigns (permission: `asset-measurement:UPDATE`)
- **DELETE** - Delete campaigns (permission: `asset-measurement:DELETE`)
- **SUPER_ADMIN** - Bypass all permission checks
