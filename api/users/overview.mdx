## Overview

The User Management API provides endpoints to create, read, update, and delete users within the SMyEG geospatial monitoring system. All endpoints enforce organization-level isolation and role-based access control.

## Authentication

All user endpoints require authentication. Users must have appropriate permissions:
- **ADMIN** or **SUPER_ADMIN** roles have full access
- Other users need specific permissions: `users:READ`, `users:CREATE`, `users:UPDATE`, `users:DELETE`

## Organization Isolation

- **SUPER_ADMIN**: Can view and manage users across all organizations by passing `organizationId` query parameter
- **ADMIN**: Can only manage users within their own organization
- Regular users: Can only view/edit users within their organization scope

## List Users

<CodeGroup>
```bash GET /api/users
curl -X GET "https://api.smyeg.com/api/users?page=1&limit=25&search=john&status=ACTIVE&role=ADMIN" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

Retrieves a paginated list of users with optional filtering.

### Query Parameters

<ParamField query="page" type="integer" default={1}>
  Page number for pagination (minimum: 1)
</ParamField>

<ParamField query="limit" type="integer" default={25}>
  Number of items per page (minimum: 1, maximum: 100)
</ParamField>

<ParamField query="search" type="string">
  Search term to filter users by email, first name, or last name (case-insensitive)
</ParamField>

<ParamField query="status" type="enum">
  Filter by user status
  <ul>
    <li>`ACTIVE` - User is active</li>
    <li>`INACTIVE` - User is inactive</li>
    <li>`PENDING_VERIFICATION` - User awaiting verification</li>
    <li>`LOCKED` - User account is locked</li>
    <li>`DELETED` - User is deleted</li>
  </ul>
</ParamField>

<ParamField query="role" type="string">
  Filter by role slug (e.g., "ADMIN", "USER", "VIEWER")
</ParamField>

<ParamField query="sortBy" type="enum">
  Field to sort by
  <ul>
    <li>`email` - Sort by email</li>
    <li>`status` - Sort by status</li>
    <li>`createdAt` - Sort by creation date</li>
  </ul>
</ParamField>

<ParamField query="sortOrder" type="enum" default="desc">
  Sort order: `asc` or `desc`
</ParamField>

<ParamField query="organizationId" type="string">
  (SUPER_ADMIN only) Filter users by organization ID
</ParamField>

## Create User

<CodeGroup>
```bash POST /api/users
curl -X POST "https://api.smyeg.com/api/users" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "jane.smith@example.com",
    "firstName": "Jane",
    "lastName": "Smith",
    "roleSlug": "VIEWER",
    "organizationId": "660e8400-e29b-41d4-a716-446655440001",
    "password": "SecurePass123"
  }'
```
</CodeGroup>

Creates a new user in the system.

### Required Permissions

- **ADMIN** or **SUPER_ADMIN** role, OR
- `users:CREATE` permission

### Request Body

<ParamField body="email" type="string" required>
  Valid email address (must be unique)
</ParamField>

<ParamField body="firstName" type="string">
  User's first name (minimum 2 characters)
</ParamField>

<ParamField body="lastName" type="string">
  User's last name (minimum 2 characters)
</ParamField>

<ParamField body="roleSlug" type="string" required>
  Role slug to assign (minimum 2 characters). Examples: `ADMIN`, `VIEWER`, `EDITOR`
</ParamField>

<ParamField body="organizationId" type="string" required>
  UUID of the organization (required)
</ParamField>

<ParamField body="password" type="string">
  User password (minimum 8 characters). If omitted, a random password is generated and user status is set to `PENDING_VERIFICATION`
</ParamField>

### Response

Returns the created user object with status code `201`.

<ResponseField name="id" type="string" required>
  UUID of the created user
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address
</ResponseField>

<ResponseField name="firstName" type="string">
  User's first name
</ResponseField>

<ResponseField name="lastName" type="string">
  User's last name
</ResponseField>

<ResponseField name="status" type="enum" required>
  User status: `ACTIVE` (if password provided) or `PENDING_VERIFICATION` (if no password)
</ResponseField>

<ResponseField name="organizationId" type="string" required>
  UUID of the user's organization
</ResponseField>

<ResponseField name="organization" type="object">
  Organization details
</ResponseField>

<ResponseField name="userRoles" type="UserRole[]" required>
  Array containing the assigned role
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of user creation
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of last update
</ResponseField>

## Data Models

### User Model (Prisma Schema)

```typescript
model User {
  id                     String                   @id @default(uuid()) @db.Uuid
  organizationId         String?                  @db.Uuid
  email                  String                   @unique @db.VarChar(255)
  emailVerifiedAt        DateTime?
  username               String?                  @unique @db.VarChar(100)
  passwordHash           String?                  @db.VarChar(255)
  authProvider           AuthProvider             @default(LOCAL)
  providerId             String?                  @db.VarChar(255)
  status                 UserStatus               @default(PENDING_VERIFICATION)
  firstName              String?                  @db.VarChar(100)
  lastName               String?                  @db.VarChar(100)
  displayName            String?                  @db.VarChar(200)
  avatarUrl              String?                  @db.VarChar(500)
  phone                  String?                  @db.VarChar(20)
  phoneVerifiedAt        DateTime?
  timezone               String?                  @default("UTC") @db.VarChar(50)
  locale                 String?                  @default("en") @db.VarChar(10)
  lastLoginAt            DateTime?
  lastLoginIp            String?                  @db.VarChar(45)
  failedLoginAttempts    Int                      @default(0)
  lockedUntil            DateTime?
  mfaEnabled             Boolean                  @default(false)
  mfaSecret              String?                  @db.VarChar(255)
  mfaBackupCodes         String[]
  metadata               Json?
  createdAt              DateTime                 @default(now())
  updatedAt              DateTime                 @updatedAt
  deletedAt              DateTime?
  organization           Organization?            @relation(fields: [organizationId], references: [id], onDelete: SetNull)
  userRoles              UserRole[]
}
```

  DELETED

## Audit Trail

All user creation operations are logged in the `AuditLog` table with:
- Action: `CREATE`
- Entity Type: `User`
- Entity ID: Created user's UUID
- User ID: ID of the user who performed the action
- New Values: Email of the created user
