## Overview

ForestPatrimonyLevel4 represents the operational units within the forest patrimony hierarchy - stands (RODAL), plots (PARCELA), enumeration units, management units (UNIDAD_DE_MANEJO), smallholdings (CONUCO), or other land uses (OTRO_USO).

<Info>
  Level 4 units are the primary operational level where:
  - **Geospatial data** is stored (polygons with coordinates)
  - **Land use** is tracked and monitored
  - **Biological assets** (plantations) are registered
  - **Patrimonial variations** are recorded
</Info>

## Hierarchy

```
ForestPatrimonyLevel3 (Parent)
└── ForestPatrimonyLevel4 ← Current Level (with geometry)
    ├── ForestGeometryN4 (MultiPolygon geometry)
    ├── ForestPatrimonyLevel5 (Child plots/samples)
    └── ForestBiologicalAssetLevel6 (Plantations)
```

## Enum: PatrimonyLevel4Type

Supported level 4 types:

| Value | Description |
|-------|-------------|
| `RODAL` | Stand |
| `PARCELA` | Plot |
| `ENUMERATION` | Enumeration unit |
| `UNIDAD_DE_MANEJO` | Management unit |
| `CONUCO` | Smallholding/Garden plot |
| `OTRO_USO` | Other use |

## Enum: FscCertificateStatus

FSC certification status:

| Value | Description |
|-------|-------------|
| `SI` | FSC certified |
| `NO` | Not FSC certified |

#### Automatic Processing

When creating a Level 4 unit with geometry, the system automatically:

1. **Validates land use** against the LandUseType catalog
2. **Checks for overlaps** with other active Level 4 geometries
3. **Converts geometry** to PostGIS MultiPolygon (SRID 4326)
4. **Calculates metrics** (area in hectares, centroid coordinates)
5. **Deactivates old geometries** if updating an existing Level 4
6. **Creates a patrimonial variation record** for accounting
7. **Enqueues a recalc job** to update parent aggregations

<Warning>
  If the polygon overlaps with another active Level 4 geometry, the request will fail with an error identifying the conflicting unit.
</Warning>

#### Response

Returns the created Level 4 unit object with HTTP status 201.

<Info>

  - Creates or updates Level 4 units

  -H "Authorization: Bearer YOUR_TOKEN" \

</CodeGroup>

<ParamField formData="file" type="file" required>
  ZIP file containing shapefile components (.shp, .shx, .dbf, .prj)
</ParamField>

<ParamField formData="variationDate" type="date" optional>

</ParamField>

#### Shapefile Attribute Requirements

The shapefile must include these attribute fields (case-insensitive):

| Attribute | Alternatives | Required | Description |
|-----------|--------------|----------|-------------|
| nivel2 | nivel_2, cod_n2, n2 | Yes | Level 2 code |
| nivel3 | nivel_3, cod_n3, n3 | Yes | Level 3 code |
| nivel4 | nivel_4, cod_n4, n4 | Yes | Level 4 code |
| usoactual | uso_actual, currentlandusename | No | Current land use name |
| usoanterior | uso_anterior, previouslandusename | No | Previous land use name |
| fechavariacion | fecha_variacion, variationdate | No | Variation date (overrides form parameter) |
| observaciones | observacion, notas, variationnotes | No | Variation notes (overrides form parameter) |

#### Import Process

The import runs asynchronously:

1. **PENDING**: Job created, file uploaded
2. **EXTRACTING**: Reading shapefile from ZIP
3. **VALIDATING**: Checking required fields and structure
4. **PROCESSING**: Processing each feature:
   - Validates hierarchy (Level 2 → Level 3 → Level 4)
   - Validates land use names against catalog
   - Converts Polygon/MultiPolygon to PostGIS format
   - Deactivates old geometries
   - Inserts new geometry version
   - Updates Level 4 metrics (area, centroid)
   - Creates patrimonial variation record
5. **COMPLETED**: All features processed successfully
6. **FAILED**: Import failed with error message

## Geospatial Data

### Geometry Versioning

Level 4 geometries are versioned in the `forest_geometry_n4` table:

- Each geometry update creates a **new version**
- Old versions are marked as `isActive = false` and get a `validTo` timestamp
- Only one geometry per Level 4 unit can be active at a time
- Historical geometries are preserved for audit trail

### Geometry Format

- **Storage**: PostGIS MultiPolygon (SRID 4326)
- **API Input**: GeoJSON Polygon or MultiPolygon
- **API Output**: GeoJSON MultiPolygon
- **Coordinate System**: WGS84 (EPSG:4326)

### Calculated Fields

When a geometry is added or updated:

- `totalAreaHa` - Calculated from ST_Area in hectares
- `centroidLatitude` - Y coordinate of ST_Centroid
- `centroidLongitude` - X coordinate of ST_Centroid
- `superficieHa` - Stored in geometry record (12,4 precision)
