## Overview

ForestPatrimonyLevel2 represents the highest level of land organization in the forest patrimony hierarchy. These are typically large land units like farms (FINCA), properties (PREDIO), ranches (HATO), estates (HACIENDA), or protected areas (ABRAE).

<Info>
  Level 2 units are the root of the forest patrimony hierarchy. They contain Level 3 units (compartments, blocks, zones) which in turn contain Level 4 units (stands, plots).
</Info>

## Hierarchy

```
Organization
└── ForestPatrimonyLevel2 (FINCA, PREDIO, HATO, FUNDO, HACIENDA, ABRAE)
    └── ForestPatrimonyLevel3 (COMPARTIMIENTO, LOTE, ZONA, BLOQUE)
        └── ForestPatrimonyLevel4 (RODAL, PARCELA, UNIDAD_DE_MANEJO)
            └── ForestPatrimonyLevel5 (REFERENCIA, SUBUNIDAD, SUBPARCELA)
```

## Enum: PatrimonyLevel2Type

Supported level 2 types:

| Value | Description |
|-------|-------------|
| `FINCA` | Farm |
| `PREDIO` | Property |
| `HATO` | Ranch |
| `FUNDO` | Estate |
| `HACIENDA` | Hacienda |
| `ABRAE` | Protected natural area |

## Enum: LegalStatus

Supported legal status values:

| Value | Description |
|-------|-------------|
| `ADQUISICION` | Acquisition |
| `ARRIENDO` | Lease |
| `USUFRUCTO` | Usufruct |
| `COMODATO` | Loan for use |
| `DECRETO` | Decree |

### Create Level 2 Unit

<CodeGroup>
```bash cURL
curl -X POST "https://api.smyeg.com/api/forest/patrimony" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "level": "2",
    "data": {
      "code": "FINCA-001",
      "name": "Finca El Bosque",
      "type": "FINCA",
      "totalAreaHa": 1250.50,
      "legalStatus": "ADQUISICION",
      "isActive": true
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.smyeg.com/api/forest/patrimony', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    level: '2',
    data: {
      code: 'FINCA-001',
      name: 'Finca El Bosque',
      type: 'FINCA',
      totalAreaHa: 1250.50,
      legalStatus: 'ADQUISICION',
      isActive: true
    }
  })
});
const data = await response.json();
```
</CodeGroup>

### Update Level 2 Unit

<CodeGroup>
```bash cURL
curl -X PATCH "https://api.smyeg.com/api/forest/patrimony" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "level": "2",
    "id": "uuid-here",
    "data": {
      "name": "Finca El Bosque Actualizado",
      "totalAreaHa": 1300.00
    }
  }'
```
</CodeGroup>

#### Request Body

<ParamField body="level" type="string" required>
  Must be `"2"`
</ParamField>

<ParamField body="id" type="string" required>
  UUID of the Level 2 unit to update
</ParamField>

<ParamField body="data" type="object" required>
  Object with fields to update (partial update supported)
</ParamField>

### Delete Level 2 Unit

<Warning>
  A Level 2 unit cannot be deleted if it has child Level 3 units. Delete all child units first.
</Warning>

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.smyeg.com/api/forest/patrimony" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "level": "2",
    "id": "uuid-here"
  }'
```
</CodeGroup>

## Import Level 2 Units

### Bulk Import from CSV/Excel

<CodeGroup>
```bash cURL
curl -X POST "https://api.smyeg.com/api/forest/patrimony/import" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@patrimony_level2.xlsx" \
  -F "level=2"
```
</CodeGroup>

#### Form Data

<ParamField formData="file" type="file" required>
  CSV or XLSX file with Level 2 data
</ParamField>

<ParamField formData="level" type="string" required>
  Must be `"2"`
</ParamField>

#### Required Columns

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| code | string | Yes | Unique unit code |
| name | string | Yes | Unit name |
| type | enum | Yes | Level 2 type |
| totalAreaHa | decimal | Yes | Total area in hectares |
| legalStatus | enum | No | Legal status |
| isActive | boolean | No | Active status (default: true) |

<Info>
  The import process supports both creation and update:
  - If a unit with the same code exists, it will be updated
  - If no unit exists with that code, a new one will be created
</Info>
