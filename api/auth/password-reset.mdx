## Overview

SMyEG provides a secure two-step password reset process:

1. **Request Reset Token** - User requests a password reset via email
2. **Reset Password** - User submits new password with the token

This flow ensures secure password recovery while invalidating all existing sessions.

## Step 1: Request Reset Token

Initiate the password reset process by requesting a reset token.

### Endpoint

```
POST /api/auth/forgot-password
```

### Request Body

<ParamField body="email" type="string" required>
  Email address of the account to reset.
  
  **Validation:** Must be a valid email format.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Always `true` for valid requests
</ResponseField>

<ResponseField name="data" type="object">
  <ResponseField name="message" type="string">
    Generic success message (same whether email exists or not)
  </ResponseField>
</ResponseField>

### Example Request

```bash cURL
curl -X POST https://api.smyeg.example/api/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com"
  }'
```

```javascript JavaScript
const response = await fetch('/api/auth/forgot-password', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'user@example.com',
  }),
});

const data = await response.json();
```

### Success Response (200)

```json
{
  "success": true,
  "data": {
    "message": "Si el correo existe, se enviará un enlace de recuperación."
  }
}
```

<Note>
  The API returns the same response whether or not the email exists. This prevents email enumeration attacks.
</Note>

### Error Response (400)

```json
{
  "success": false,
  "error": "Datos inválidos",
  "details": {
    "fieldErrors": {
      "email": ["Invalid email"]
    },
    "formErrors": []
  }
}
```

Returned when the email format is invalid.

### Reset Token Details

If the email exists in the system, a reset token is generated with:
- **Token:** Cryptographically secure random token
- **Expiration:** 1 hour from generation
- **Storage:** Token is hashed before database storage
- **Single Use:** Token is marked as used after successful reset

<Warning>
  In a production environment, the reset token should be sent via email. The current implementation generates the token but email delivery must be configured separately.
</Warning>

## Step 2: Reset Password

Complete the password reset using the token received via email.

### Endpoint

```
POST /api/auth/reset-password
```

### Request Body

<ParamField body="token" type="string" required>
  The reset token received via email.
  
  **Validation:** Must be a non-empty string.
</ParamField>

<ParamField body="password" type="string" required>
  New password for the account.
  
  **Validation:**
  - Minimum 8 characters
  - At least one uppercase letter (A-Z)
  - At least one lowercase letter (a-z)
  - At least one number (0-9)
</ParamField>

<ParamField body="confirmPassword" type="string" required>
  Password confirmation. Must match the `password` field exactly.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the password reset was successful
</ResponseField>

<ResponseField name="data" type="object">
  <ResponseField name="message" type="string">
    Confirmation message
  </ResponseField>
</ResponseField>

### Example Request

```bash cURL
curl -X POST https://api.smyeg.example/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "token": "abc123def456...",
    "password": "NewSecurePass123",
    "confirmPassword": "NewSecurePass123"
  }'
```

```javascript JavaScript
const response = await fetch('/api/auth/reset-password', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    token: 'abc123def456...',
    password: 'NewSecurePass123',
    confirmPassword: 'NewSecurePass123',
  }),
});

const data = await response.json();
```

### Success Response (200)

```json
{
  "success": true,
  "data": {
    "message": "Contraseña actualizada correctamente"
  }
}
```

### Error Responses

#### Validation Error (400)

```json
{
  "success": false,
  "error": "Datos inválidos",
  "details": {
    "fieldErrors": {
      "password": ["Debe contener al menos una mayúscula"],
      "confirmPassword": ["Las contraseñas no coinciden"]
    },
    "formErrors": []
  }
}
```

Returned when request body fails validation.

**Common validation errors:**

| Field | Error | Reason |
|-------|-------|--------|
| `token` | "Required" | Token is missing |
| `password` | "String must contain at least 8 character(s)" | Password too short |
| `password` | "Debe contener al menos una mayúscula" | Missing uppercase letter |
| `password` | "Debe contener al menos una minúscula" | Missing lowercase letter |
| `password` | "Debe contener al menos un número" | Missing number |
| `confirmPassword` | "Las contraseñas no coinciden" | Passwords don't match |

#### Invalid or Expired Token (400)

```json
{
  "success": false,
  "error": "Token inválido o expirado"
}
```

Returned when:
- Token doesn't exist in the database
- Token has already been used
- Token expiration time has passed (> 1 hour old)

## Password Reset Process

When a password is successfully reset, the following actions occur atomically:

1. **Password Update** - New password is hashed and stored
2. **Reset Counters** - Failed login attempts counter is reset to 0
3. **Clear Lock** - Account lock (if any) is removed
4. **Mark Token Used** - Reset token is marked with `usedAt` timestamp
5. **Revoke Sessions** - All active sessions are invalidated
6. **Audit Log** - A PASSWORD_RESET audit log entry is created

All these operations happen in a database transaction to ensure consistency.

## Validation Schemas

The password reset endpoints use Zod for validation:

### Forgot Password Schema

```typescript
import { z } from "zod";

const forgotPasswordSchema = z.object({
  email: z.string().email(),
});
```

### Reset Password Schema

```typescript
import { z } from "zod";

const resetPasswordSchema = z
  .object({
    token: z.string().min(1),
    password: z
      .string()
      .min(8)
      .regex(/[A-Z]/, "Debe contener al menos una mayúscula")
      .regex(/[a-z]/, "Debe contener al menos una minúscula")
      .regex(/[0-9]/, "Debe contener al menos un número"),
    confirmPassword: z.string(),
  })
  .refine((data) => data.password === data.confirmPassword, {
    message: "Las contraseñas no coinciden",
    path: ["confirmPassword"],
  });
```

## Security Considerations

### Token Security

- **Secure Generation:** Tokens are generated using cryptographically secure random functions
- **Hashed Storage:** Tokens are hashed before storage (only hash is stored in database)
- **Time-Limited:** Tokens expire after 1 hour
- **Single Use:** Tokens are marked as used after successful reset

### Session Invalidation

When a password is reset:
- All active sessions for that user are revoked immediately
- Users must log in again with their new password
- This ensures compromised sessions cannot be used after password reset

### Audit Trail

Password resets are logged with:
- `action: "PASSWORD_RESET"`
- `entityType: "User"`
- `entityId`: User ID
- Timestamp

### Email Enumeration Prevention

The forgot password endpoint returns the same response whether the email exists or not. This prevents attackers from using the endpoint to discover valid email addresses.

### Account Recovery

Password reset also:
- Clears the failed login attempts counter
- Removes any account locks
- Allows locked users to regain access

## Complete Flow Example

```javascript
// Step 1: Request reset token
const forgotResponse = await fetch('/api/auth/forgot-password', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'user@example.com',
  }),
});

// User receives email with token (email delivery not shown)

// Step 2: Reset password with token
const resetResponse = await fetch('/api/auth/reset-password', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    token: 'abc123def456...', // From email
    password: 'NewSecurePass123',
    confirmPassword: 'NewSecurePass123',
  }),
});

const result = await resetResponse.json();
if (result.success) {
  console.log('Password reset successful!');
  // Redirect to login page
}
```
