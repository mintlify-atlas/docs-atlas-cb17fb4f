This comprehensive guide covers installing and configuring SMyEG for development and production environments.

## System Requirements

### Software Requirements

- **Node.js**: Version 20 or higher
- **npm** or **pnpm**: Latest stable version
- **PostgreSQL**: Version 15 or higher with PostGIS extension
- **Redis**: Version 7 or higher (for session management and caching)
- **Docker**: Optional, but recommended for local development

### Hardware Requirements

**Minimum (Development):**
- 4 GB RAM
- 2 CPU cores
- 10 GB disk space

**Recommended (Production):**
- 8+ GB RAM
- 4+ CPU cores
- 50+ GB disk space (depends on geospatial data volume)

## Installation Methods

<Info>
  Choose the installation method that best fits your environment. Docker Compose is recommended for quick setup.
</Info>

### Method 1: Docker Compose (Recommended)

<Steps>
  <Step title="Install Dependencies">
    Install Node.js dependencies:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Start Services">
    Launch PostgreSQL and Redis containers:

    ```bash
    docker compose up -d
    ```

    This starts:
    - PostgreSQL 15 (Alpine) on port `5432`
    - Redis 7 (Alpine) on port `6379`

    Verify containers are running:

    ```bash
    docker ps
    ```

    You should see both `postgres` and `redis` containers in "Up" status.
  </Step>
</Steps>

### Method 2: Local Installation

If you prefer to install PostgreSQL and Redis locally without Docker:

<Steps>
  <Step title="Install PostgreSQL">
    **Ubuntu/Debian:**
    ```bash
    sudo apt update
    sudo apt install postgresql-15 postgresql-contrib postgis
    ```

    **macOS (Homebrew):**
    ```bash
    brew install postgresql@15 postgis
    brew services start postgresql@15
    ```

    **Windows:**
    Download and install from [PostgreSQL Downloads](https://www.postgresql.org/download/windows/)
  </Step>

  <Step title="Install Redis">
    **Ubuntu/Debian:**
    ```bash
    sudo apt install redis-server
    sudo systemctl start redis-server
    ```

    **macOS (Homebrew):**
    ```bash
    brew install redis
    brew services start redis
    ```

    **Windows:**
    Download from [Redis for Windows](https://github.com/microsoftarchive/redis/releases)
  </Step>

  <Step title="Create Database">
    Create the application database:

    ```bash
    psql -U postgres
    ```

    ```sql
    CREATE DATABASE app_dev;
    CREATE USER app_user WITH PASSWORD 'your_password';
    GRANT ALL PRIVILEGES ON DATABASE app_dev TO app_user;
    \q
    ```
  </Step>
</Steps>

## Environment Configuration

Create a `.env` file in the project root with the following variables:

### Essential Variables

```bash .env
# ============================================
# Database Configuration
# ============================================
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/app_dev"

# ============================================
# NextAuth Configuration
# ============================================
NEXTAUTH_SECRET="your-secret-key-here-change-in-production"
NEXTAUTH_URL="http://localhost:3000"

# ============================================
# Application Configuration
# ============================================
NEXT_PUBLIC_APP_URL="http://localhost:3000"
NODE_ENV="development"
```

### Optional Variables

```bash .env
# ============================================
# Geospatial Worker Configuration
# ============================================
GEO_WORKER_INTERVAL_MS="4000"          # Polling interval (default: 4000ms)
GEO_IMPORT_BATCH_SIZE="5"              # Import jobs per cycle (default: 5)
GEO_RECALC_BATCH_SIZE="10"             # Recalc jobs per cycle (default: 10)
GEO_VARIATION_BATCH_SIZE="15"          # Variation jobs per cycle (default: 15)
GEO_WORKER_RUN_ONCE="false"            # Run once and exit (default: false)
GEO_WORKER_SECRET="your-worker-secret" # Worker authentication secret

# ============================================
# Asset Measurement Worker Configuration
# ============================================
ASSET_MEASUREMENT_WORKER_INTERVAL_MS="4000"  # Polling interval (default: 4000ms)
ASSET_MEASUREMENT_WORKER_BATCH_SIZE="20"     # Jobs per cycle (default: 20)
ASSET_MEASUREMENT_WORKER_RUN_ONCE="false"    # Run once and exit (default: false)
ASSET_MEASUREMENT_WORKER_SECRET="your-secret" # Worker authentication secret

# ============================================
# Export Limits
# ============================================
USERS_EXPORT_MAX_LIMIT="10000"
ROLES_EXPORT_MAX_LIMIT="5000"
ORGANIZATIONS_EXPORT_MAX_LIMIT="5000"
FOREST_PATRIMONY_EXPORT_MAX_LIMIT="50000"
FOREST_BIOLOGICAL_ASSET_EXPORT_MAX_LIMIT="100000"
FOREST_CONFIG_EXPORT_MAX_LIMIT="10000"
GENERAL_CONFIG_EXPORT_MAX_LIMIT="10000"
MEASUREMENT_CAMPAIGNS_EXPORT_MAX_LIMIT="50000"
MEASUREMENT_OBSERVATIONS_EXPORT_MAX_LIMIT="100000"
```

<Warning>
  **Security Best Practices:**
  - Never commit `.env` files to version control
  - Generate strong random values for all secrets
  - Use different secrets for development and production
  - Rotate secrets regularly in production
</Warning>

### Generating Secure Secrets

Use OpenSSL to generate secure random secrets:

```bash
openssl rand -base64 32
```

Or use Node.js:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

## Database Setup

<Steps>
  <Step title="Generate Prisma Client">
    Generate the type-safe Prisma database client:

    <CodeGroup>
      ```bash npm
      npm run db:generate
      ```

      ```bash pnpm
      pnpm db:generate
      ```
    </CodeGroup>

    This reads your Prisma schema from `prisma/schema.prisma` and generates the client at `node_modules/@prisma/client`.
  </Step>

  <Step title="Run Migrations">
    Apply all database migrations to create tables, extensions, and triggers:

    <CodeGroup>
      ```bash npm
      npm run db:migrate
      ```

      ```bash pnpm
      pnpm db:migrate
      ```
    </CodeGroup>

    This creates:
    - All application tables (users, organizations, roles, forest patrimony, etc.)
    - PostGIS spatial extensions for geometry storage
    - Audit log partitioning by month
    - Database triggers for automated workflows
  </Step>

  <Step title="Seed Initial Data">
    Populate the database with initial system data:

    <CodeGroup>
      ```bash npm
      npm run db:seed
      ```

      ```bash pnpm
      pnpm db:seed
      ```
    </CodeGroup>

    The seed script creates:
    - **Default admin user** (`admin@example.com` / `Admin1234`)
    - **System roles** (Super Admin, Admin, User)
    - **Modules and permissions** (Authentication, Users, Organizations, Forest Patrimony, Assets)
    - **Sample geographic data** (continents, countries, regions)
  </Step>

  <Step title="Verify Database Setup">
    Connect to the database and verify tables were created:

    ```bash
    psql postgresql://postgres:postgres@localhost:5432/app_dev
    ```

    ```sql
    -- List all tables
    \dt

    -- Check user count
    SELECT COUNT(*) FROM users;

    -- Check PostGIS extension
    SELECT PostGIS_Version();

    -- Exit
    \q
    ```
  </Step>
</Steps>

## Background Workers

SMyEG uses background workers for asynchronous processing of geospatial data and asset measurements.

### Geospatial Worker

Processes shapefile imports, geometry recalculations, and land variations:

<CodeGroup>
  ```bash npm
  npm run worker:geo
  ```

  ```bash pnpm
  pnpm worker:geo
  ```
</CodeGroup>

**Configuration:**
- `GEO_WORKER_INTERVAL_MS` - Polling interval (default: 4000ms)
- `GEO_IMPORT_BATCH_SIZE` - Import jobs per cycle (default: 5)
- `GEO_RECALC_BATCH_SIZE` - Recalculation jobs per cycle (default: 10)
- `GEO_VARIATION_BATCH_SIZE` - Variation jobs per cycle (default: 15)

### Asset Measurement Worker

Processes biological, hydrological, and dendrometric asset calculations:

<CodeGroup>
  ```bash npm
  npm run worker:assets
  ```

  ```bash pnpm
  pnpm worker:assets
  ```
</CodeGroup>

**Configuration:**
- `ASSET_MEASUREMENT_WORKER_INTERVAL_MS` - Polling interval (default: 4000ms)
- `ASSET_MEASUREMENT_WORKER_BATCH_SIZE` - Jobs per cycle (default: 20)

### Running Workers with PM2 (Production)

For production deployments on Windows or Linux, use PM2 to manage workers:

<Steps>
  <Step title="Install PM2">
    ```bash
    npm install -g pm2
    ```
  </Step>

  <Step title="Build Application">
    <CodeGroup>
      ```bash npm
      npm run build
      ```

      ```bash pnpm
      pnpm build
      ```
    </CodeGroup>
  </Step>

  <Step title="Start All Processes">
    Start web server and workers defined in `ecosystem.config.cjs`:

    ```bash
    pm2 start ecosystem.config.cjs
    ```
  </Step>

  <Step title="Monitor Processes">
    View status and logs:

    ```bash
    # Check process status
    pm2 status

    # View web server logs
    pm2 logs confor-web

    # View geo worker logs
    pm2 logs confor-geo-worker
    ```
  </Step>

  <Step title="Enable Auto-Restart">
    Configure PM2 to restart processes on reboot:

    ```bash
    # Save current process list
    pm2 save

    # Generate startup script
    pm2 startup

    # Run the command shown by pm2 startup, then save again
    pm2 save
    ```
  </Step>
</Steps>

**Useful PM2 Commands:**

```bash
# Restart specific process
pm2 restart confor-web
pm2 restart confor-geo-worker

# Stop all processes
pm2 stop all

# Delete all processes
pm2 delete all

# View detailed process info
pm2 info confor-web
```

## Database Maintenance

### Creating Audit Log Partitions

SMyEG uses monthly partitions for audit logs. Create the next partition:

```bash
node scripts/create-audit-partition.ts
```

Or schedule it monthly with cron:

```bash
0 0 1 * * cd /path/to/app && node scripts/create-audit-partition.ts
```

### Backup Database

Create a database backup:

```bash
pg_dump -U postgres -d app_dev -F c -f backup_$(date +%Y%m%d).dump
```

### Restore Database

Restore from a backup:

```bash
pg_restore -U postgres -d app_dev -c backup_20260313.dump
```

## Troubleshooting

### Database Migration Errors

**Problem:** Migration fails with "relation already exists"

**Solution:**
```bash
# Reset database (WARNING: deletes all data)
npm run db:push
```

### PostGIS Extension Missing

**Problem:** Spatial queries fail

**Solution:**
```sql
CREATE EXTENSION IF NOT EXISTS postgis;
```

### Port Conflicts

**Problem:** Port 3000, 5432, or 6379 already in use

**Solution:**
```bash
# Change Next.js port
npm run dev -- -p 3001

# Change Docker ports in docker-compose.yml
ports:
  - "5433:5432"  # PostgreSQL
  - "6380:6379"  # Redis
```

### Prisma Client Out of Sync

**Problem:** Type errors or "Prisma Client is not configured" error

**Solution:**
```bash
npm run db:generate
```

### Worker Authentication Failed

**Problem:** Workers cannot authenticate with API

**Solution:** Set worker secrets in `.env`:
```bash
GEO_WORKER_SECRET="your-secret-here"
ASSET_MEASUREMENT_WORKER_SECRET="your-secret-here"
```

### Out of Memory During Build

**Problem:** Build fails with heap out of memory

**Solution:**
```bash
NODE_OPTIONS="--max-old-space-size=4096" npm run build
```
